R package · phylogenetic diversification

Inference of diversification with endogenous drivers

Monte Carlo conditional-likelihood inference for birth–death models whose speciation and extinction rates depend on covariates the diversification process generates itself.

Install GitHub Usage

Given a dated phylogeny of the surviving species of a clade, emphasis estimates the speciation and extinction rates that generated it — and lets those rates depend on the state of the clade itself: how many lineages there are, and how isolated each lineage is relative to its contemporaries.

A reconstructed molecular phylogeny records only the lineages that survive to the present. The lineages that went extinct are exactly what carries the information about the extinction rate, so the likelihood of the observed tree is an integral over all possible histories of unobserved lineages. emphasis evaluates and maximizes that likelihood by data-augmentation importance sampling embedded in a Monte Carlo EM.

The model

Time runs forward from the crown (age \(0\)) to the present (age \(T\)). At any moment the clade is a set of living lineages, each carrying an instantaneous speciation rate \(\lambda_s(t)\) and extinction rate \(\mu_s(t)\) built from a linear predictor over covariates, passed through a link:

\[ \eta_\lambda = \beta_0 + \beta_N N + \beta_D D, \qquad \eta_\mu = \gamma_0 + \gamma_N N + \gamma_D D. \]

Covariates — an orthogonal basis

Let \(t_s\) be when lineage \(s\) last speciated and \(I_s(t) = t - t_s\) its isolation time. Their clade total is the pendant phylogenetic diversity \(P = \sum_s I_s\). Because \(P = N\bar I\) — richness times mean isolation — using \(P\) directly makes it collinear with richness, so it is split into a global mean and a local deviation:

\[ M(t) = \frac{P(t)}{N(t)} = \bar I(t), \qquad D_s(t) = I_s(t) - M(t), \qquad \sum_s D_s(t) = 0. \]

Because \(\sum_s D_s = 0\) at every instant, \(D\) is orthogonal to the diversity \(N\): the two covariates measure distinct effects — how many lineages there are (\(N\)), and whether a lineage is older or younger than its contemporaries (\(D\)). \(N\) reads the branching times; \(D\) reads the topology. A model activates a subset, given as a shortcut or a formula:

ShortcutActiveParametersHypothesis
"cr"none\((\beta_0, \gamma_0)\)constant rates (null)
"dd"\(N\)\((\beta_0, \beta_N, \gamma_0, \gamma_N)\)diversity dependence
"d"\(D\)\((\beta_0, \beta_D, \gamma_0, \gamma_D)\)age-dependent speciation
"nd"\(N + D\)\((\beta_0, \beta_N, \beta_D, \gamma_0, \gamma_N, \gamma_D)\)both (recommended)

The recommended model is ~ N + D (6 parameters); ep is a legacy alias for d. See the biological meaning of D for what \(\beta_D\) tests (age-dependent speciation).

Link functions

The covariate predictor \(\eta_{\text{cov}}\) is mapped to a non-negative rate, selected with link = "linear" | "exponential" | "gaussian":

LinkRateBehaviour
Linear\(\max(0,\ \beta_0 + \eta_{\text{cov}})\)clipped at zero; can be exactly zero
Exponential\(\exp(\beta_0 + \eta_{\text{cov}})\)strictly positive; log-linear
Gaussian\(\beta_0 \cdot \exp\!\big(-(\eta_{\text{cov}} - 1)^2 / 2\big)\)bounded, bell-shaped

The likelihood

Write the observed reconstructed tree as \(y_{\text{obs}}\) and the latent extinct and unsampled lineages as \(z\). The complete configuration has a closed-form density \(f_\theta(y_{\text{obs}}, z)\); only \(y_{\text{obs}}\) is observed, so the likelihood is the intractable marginal

\[ L_\theta(y_{\text{obs}}) = \int f_\theta(y_{\text{obs}}, z)\, dz. \]

For a single augmented tree — a piecewise-homogeneous birth–death realization — the complete-data log-density is a log-rate for every event that happened, minus the integrated total hazard:

\[ \log f_\theta = \sum_{\text{spec}} \log \lambda(t_b) + \sum_{\text{ext}} \log \mu(t_e) - \int_0^T N(t)\big[\lambda(t) + \mu(t)\big]\, dt \;+\; S_\rho, \]

where \(S_\rho\) is the incomplete-sampling correction (below). When the deviation covariate \(D\) is active each lineage has its own rate; because \(D = (t - t_s) - M\) is constant within each inter-event segment, the hazard integral is still evaluated in closed form under the exponential and Gaussian links.

Importance sampling

The marginal is estimated by drawing latent histories from a tractable proposal and reweighting:

\[ L_\theta \approx \frac{1}{S}\sum_{i=1}^{S} w_i, \qquad w_i = \frac{f_\theta(y_{\text{obs}}, z_i)}{q_\theta(z_i \mid y_{\text{obs}})}, \qquad z_i \sim q_\theta. \]

The proposal augments the observed tree by Poisson thinning, at the intensity of speciations whose offspring do not survive as a sampled tip:

\[ \nu(t) = N(t)\,\lambda(t)\,\bigl(1 - \rho\, e^{-\mu(t)(T - t)}\bigr). \]

Monte Carlo EM

To maximize the likelihood, emphasis runs EM on the latent lineages. The objective is the expected complete-data log-likelihood,

\[ Q(\theta \mid \theta') = \mathbb{E}_{z \mid y_{\text{obs}}, \theta'}\!\big[\log f_\theta(y_{\text{obs}}, z)\big] \approx \sum_i \tilde w_i \,\log f_\theta(y_{\text{obs}}, z_i), \]
  • E-step — augment the observed tree many times in parallel, keeping augmentations with finite positive weight.
  • M-step — maximize the weighted complete-data log-likelihood over \(\theta\), optionally penalized by the survival term.

Iterate until the parameter change falls below tolerance. The importance weights also yield \(\hat f\), a log-mean-exp estimate of the observed-data log-likelihood, which drives model comparison.

Conditioning on survival & incomplete sampling

A reconstructed tree exists only because the crown lineages left surviving, sampled descendants, so the likelihood is conditioned on that event:

\[ \ell_{\text{cond}}(\theta) = \ell(\theta) - \log P_\theta(\text{survival}). \]

The survival probability has no closed form under endogenous rates; it is estimated by forward simulation and emulated with a binomial GAM for fast evaluation. When only a fraction \(\rho\) of living species is sampled, set rho: the augmentation then also inserts unsampled extant lineages, and the likelihood gains a binomial sampling factor

\[ S_\rho = n_{\text{obs}} \log \rho + n_{\text{unsamp}} \log(1 - \rho). \]

With rho = 1 the tree is completely sampled and \(S_\rho = 0\).

Estimation methods

The same likelihood machinery is driven by three optimizers, chained in one pipeline that seeds each stage from the last.

MethodIdeaRole
GAMevaluate \(\hat f\) on a Latin-hypercube grid, fit a smooth surface, optimize itfast exploration
CEMcross-entropy: evolve a particle population, keep the elite fraction, annealglobal search
MCEMMonte Carlo EM to the maximum-likelihood estimatelocal refinement

Simulation

simulate_tree() generates trees by direct (Gillespie) forward simulation, starting from the crown and tracking the two crown clades so that loss of either is recorded as tree-wide extinction. It returns the reconstructed tree (tes), the full tree with extinct lineages (tas), and a status of "done", "extinct", or "too_large".

sim <- simulate_tree(
  pars  = c(0.8, 0.02, 0.1, 0),   # beta_0, beta_N, gamma_0, gamma_N
  max_t = 10,
  model = "dd",
  link  = "gaussian"              # or "linear", "exponential"
)
sim$status      # "done", "extinct", or "too_large"
plot(sim$tes)   # reconstructed (extant-only) tree

Usage

Full pipeline (recommended)

library(emphasis); library(ape)
data(bird.orders)

# One call: bounds, GAM, CEM, MCEM
fit <- emphasis_pipeline(bird.orders, model = "dd", link = "gaussian")
fit$pars; fit$loglik; fit$AIC

Simulate, then recover

sim <- simulate_tree(pars = c(0.9, -0.01, 0.1, 0), max_t = 10,
                     model = "dd", link = "linear")

fit <- estimate_rates(sim$tes, method = "mcem", model = "dd", link = "linear",
                      init_pars = c(0.6, -0.005, 0.15, 0),
                      control = list(lower_bound = c(1e-3, -0.05, 1e-3, -0.05),
                                     upper_bound = c(3, 0.01, 3, 0.01),
                                     sample_size = 100, max_iter = 25))

Compare covariate hypotheses

models <- c("cr", "dd", "d", "nd")   # null, diversity, age-imbalance, both
select_diversification_model(bird.orders, link = "gaussian")
