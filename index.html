R package · phylogenetic diversification

Inference of diversification with endogenous drivers

Monte Carlo conditional-likelihood inference for birth–death models whose speciation and extinction rates depend on covariates the diversification process generates itself.

Install GitHub Usage

Given a dated phylogeny of the surviving species of a clade, emphasis estimates the speciation and extinction rates that generated it — and lets those rates depend on the state of the clade itself: how many lineages there are, how old they are, and how isolated each one is.

A reconstructed molecular phylogeny records only the lineages that survive to the present. The lineages that went extinct are exactly what carries the information about the extinction rate, so the likelihood of the observed tree is an integral over all possible histories of unobserved lineages. emphasis evaluates and maximizes that likelihood by data-augmentation importance sampling embedded in a Monte Carlo EM.

The model

Time runs forward from the crown (age 0) to the present (age T). At any moment the clade is a set of living lineages, each carrying an instantaneous speciation rate λs(t) and extinction rate μs(t) built from a linear predictor over covariates, passed through a link.

ηλ = β0 + βNN + βMM + βDD,    ημ = γ0 + γNN + γMM + γDD

Covariates — an orthogonal basis

Let ts be when lineage s last speciated and Is(t) = t − ts its isolation time. Their clade total is the pendant phylogenetic diversity P = Σs Is. Because P = N·Ī (richness times mean isolation), using P directly makes it collinear with richness — so it is split into a global mean and a local deviation, giving three orthogonal covariates:

CovariateDefinitionMeaning
N(t)number of living lineagesdiversity level
M(t) = P/N = Īmean pendant ageclade maturity
Ds(t) = Is − Mfocal deviation from the meanper-lineage identity

Because Σs Ds = 0 at every instant, D is orthogonal to the intercept and to the global covariates N and M: the three axes measure distinct effects — how many lineages, how old on average, and whether this lineage is older or younger than its contemporaries.

A model is a mask over these covariates, given as a shortcut, a formula, or a length-3 binary vector:

ShortcutActiveParametersHypothesis
"cr"none(β₀, γ₀)constant rates (null)
"dd"N(β₀, βN, γ₀, γN)ecological limits via richness
"ma"M(β₀, βM, γ₀, γM)regulation by clade maturity
"rd"D(β₀, βD, γ₀, γD)lineage-specific relative isolation

Covariates combine: model = ~ N + M activates two (6 parameters). The names pd and ep are accepted as aliases for ma and rd.

Link functions

The covariate predictor ηcov is mapped to a non-negative rate, selected with link = "linear" | "exponential" | "gaussian":

LinkRateBehaviour
Linearmax(0, β0 + ηcov)clipped at zero; can be exactly zero
Exponentialexp(β0 + ηcov)strictly positive; log-linear
Gaussianβ0·exp(−(ηcov − 1)²⁄2)bounded, bell-shaped; peaks at ηcov=1

The Gaussian (quadratic-exponential) link expresses a non-monotone response — for instance a diversity level that is optimal for speciation, with the rate declining on either side.

The likelihood

Write the observed reconstructed tree as yobs and the latent extinct and unsampled lineages as z. The complete configuration has a closed-form density fθ(yobs, z); only yobs is observed, so the likelihood is the intractable marginal

Lθ(yobs) = ∫ fθ(yobs, z)  dz

For a single augmented tree — a piecewise-homogeneous birth–death realization — the complete-data log-density is a log-rate for every event that happened, minus the integrated total hazard:

log fθ = ∑speclog λ + ∑extlog μ − ∫0TN(t)[λ(t)+μ(t)] dt + Sρ

When the deviation covariate D is active each lineage has its own rate; because D = (t − ts) − M is constant within each inter-event segment, the hazard integral is still evaluated in closed form under the exponential and Gaussian links.

Importance sampling

The marginal is estimated by drawing latent histories from a tractable proposal and reweighting:

Lθ ≈ 1S∑iwi,   wi = fθ(yobs, zi) ⁄ qθ(zi | yobs),  zi ∼ qθ

The proposal augments the observed tree with extinct (and, under incomplete sampling, unsampled extant) lineages by Poisson thinning, at the intensity of speciations whose offspring do not survive as a sampled tip:

ν(t) = N(t) λ(t) (1 − ρ e−μ(t)(T−t))

Monte Carlo EM

To maximize the likelihood, emphasis runs EM on the latent lineages. The objective is the expected complete-data log-likelihood,

Q(θ | θ′) = 𝔼z | yobs, θ′[ log fθ(yobs, z) ] ≈ ∑iw̃i log fθ(yobs, zi)
  • E-step — augment the observed tree many times in parallel, keeping augmentations with finite positive weight.
  • M-step — maximize the weighted complete-data log-likelihood over θ, optionally penalized by the survival term.

Iterate until the parameter change falls below tolerance. The importance weights also yield f̂, a log-mean-exp estimate of the observed-data log-likelihood, which drives model comparison.

Conditioning on survival & incomplete sampling

A reconstructed tree exists only because the crown lineages left surviving, sampled descendants, so the likelihood is conditioned on that event:

ℓcond(θ) = ℓ(θ) − log Pθ(survival)

The survival probability has no closed form under endogenous rates; it is estimated by forward simulation and emulated with a binomial GAM for fast evaluation. When only a fraction ρ of living species is sampled, set rho: the augmentation then also inserts unsampled extant lineages, and the likelihood gains a binomial sampling factor nobs log ρ + nunsamp log(1−ρ). With rho = 1 the tree is completely sampled.

Estimation methods

The same likelihood machinery is driven by three optimizers, chained in one pipeline.

MethodIdeaRole
GAMevaluate f̂ on a Latin-hypercube grid, fit a smooth surface, optimize itfast exploration
CEMcross-entropy: evolve a particle population, keep the elite fraction, annealglobal search
MCEMMonte Carlo EM to the maximum-likelihood estimatelocal refinement
Pipelineauto_bounds → GAM → CEM → MCEM, each seeding the nextrecommended workflow

Simulation

simulate_tree() generates trees by direct (Gillespie) forward simulation, starting from the crown and tracking the two crown clades so that loss of either is recorded as tree-wide extinction. It returns the reconstructed tree (tes), the full tree with extinct lineages (tas), and a status of "done", "extinct", or "too_large".

sim <- simulate_tree(
  pars  = c(0.8, 0.02, 0.1, 0),   # beta_0, beta_N, gamma_0, gamma_N
  max_t = 10,
  model = "dd",
  link  = "gaussian"              # or "linear", "exponential"
)
sim$status      # "done", "extinct", or "too_large"
plot(sim$tes)   # reconstructed (extant-only) tree

Usage

Full pipeline (recommended)

library(emphasis); library(ape)
data(bird.orders)

# One call: bounds, GAM, CEM, MCEM
fit <- emphasis_pipeline(bird.orders, model = "dd", link = "gaussian")
fit$pars; fit$loglik; fit$AIC

Simulate, then recover

sim <- simulate_tree(pars = c(0.9, -0.01, 0.1, 0), max_t = 10,
                     model = "dd", link = "linear")

fit <- estimate_rates(sim$tes, method = "mcem", model = "dd", link = "linear",
                      init_pars = c(0.6, -0.005, 0.15, 0),
                      control = list(lower_bound = c(1e-3, -0.05, 1e-3, -0.05),
                                     upper_bound = c(3, 0.01, 3, 0.01),
                                     sample_size = 100, max_iter = 25))

Compare covariate hypotheses

models <- c("cr", "dd", "ma", "rd")   # constant, diversity, maturity, deviation
fits <- lapply(models, function(m)
  emphasis_pipeline(bird.orders, model = m, link = "gaussian",
                    stages = c("bounds", "gam")))

# or let the package fit and rank them
select_diversification_model(bird.orders, link = "gaussian")
